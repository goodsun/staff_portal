## 2026-03-07 — 工房の誕生

**参加者**: マスター（goodsun）× メフィ（CCO）

### 決定1: 命名「labo-portal」

`staff_portal` を `labo_portal` にリネーム。

> 「工房という響きが好きだから」— マスター

laboratoire（仏）= 実験室・工房。  
道具を作る場所ではなく、道具を使う人間が自分らしくいられる場所。

### 決定2: 全コンテナ共通ポータル

EC2専用から、すべての docker-openclaw コンテナで動く共通設計へ。  
コンテナ固有の機能はプラグインで対応する。

### 決定3: プラグイン3層構造

Layer 1（コア）/ Layer 2（標準・設定で有効化）/ Layer 3（固有・自作）に分離。  
「混ぜるな危険」の精神を設計に組み込む。

### 決定4: staff_auth依存を断ち切る

EC2ホストOS依存の `staff_auth` を廃止。`core/auth.py` として自前実装。

### 決定5: draft/proposals フロー

AIも人間も同じRFCフローで提案・議論・承認。  
`docs/draft/proposals/` にMarkdownで提案 → PR → レビュー → 承認。
